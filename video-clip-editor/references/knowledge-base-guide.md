# 知识库学习指南

## 知识库定位

知识库（包括用户上传的同行高 GMV 视频及相关资料）只能用于**学习**，严禁抽取其中任何原文、现成表达或具体话术进入当前脚本、发布文案或 FFmpeg 指令。

## 学习目标

### 1. 黄金三秒逻辑

从知识库中提炼：
- **钩子类型**：利益点型、身份锚定型、紧迫感型、对比反差型
- **开场句式**：如何在前 3 秒内锁定用户注意力
- **情绪节奏**：如何利用情绪波动创造"不可错过"感

**学习维度**：
```
黄金三秒 = 钩子类型 + 情绪强度 + 信息密度
```

**提炼示例**：
- 知识库原文："最后50件，错过不再有！"
- 提炼逻辑：紧迫感 + 稀缺性 + 损失厌恶
- 应用方向：在当前 SRT 中寻找类似紧迫感逻辑的原句

### 2. 塑品顺序

从知识库中提炼：
- **材质/工艺**：先展示什么面料特点，如何用微距镜头强化
- **版型/设计**：如何关联身材痛点，如何展示技术细节
- **价值背书**：如何用时间跨度（穿了三年）、场景验证（哈尔滨流汗）增强可信度

**常见塑品顺序**：
```
1. 视觉冲击（颜色/版型）→ 2. 触感验证（面料）→ 3. 痛点解决（设计细节）→ 4. 价值锚定（价格对比）
```

### 3. 转场逻辑

从知识库中提炼：
- **镜头语言**：如何从一个产品特点过渡到另一个
- **时间压缩**：如何在短时间内展示多季节/多场景
- **情绪递进**：如何让每个片段的情绪逐步升温

**转场节奏**：
```
产品展示（2-4秒）→ 细节特写（1-2秒）→ 场景代入（2-3秒）→ 情绪强化（1-2秒）
```

### 4. 钩子筛选标准

从知识库中提炼：
- **高转化钩子特征**：强情绪词 + 具体利益点 + 紧迫暗示
- **低效钩子特征**：泛泛而谈、无具体数字、无情绪触发
- **人群定向**：如何通过痛点精准定位目标用户

**钩子评分矩阵**：

| 维度 | 3分 | 2分 | 1分 |
|------|-----|-----|-----|
| 情绪强度 | 强烈感叹词 | 一般描述 | 无情绪 |
| 利益具体性 | 有明确数字/比例 | 有大概范围 | 模糊表达 |
| 紧迫暗示 | 明确限时/限量 | 暗示稀缺 | 无紧迫感 |
| 人群精准度 | 直击痛点 | 部分相关 | 泛人群 |

### 5. 高转化话术模型

从知识库中提炼：
- **福利表达模型**：主体 + 动作 + 让利程度 + 承诺
- **紧迫感模型**：时间压力 + 数量限制 + 损失恐惧
- **信任建立模型**：证据 + 背书 + 保障承诺
- **决策引导模型**：问题 + 方案 + 确定性结论

**话术模型提炼示例**：

| 模型类型 | 提炼维度 | 禁止复制 |
|---------|---------|---------|
| 福利表达 | 主体（老板/品牌）+ 让利方式（送/亏本）+ 诚意表达 | 禁止复制原文 |
| 紧迫感 | 触发词（错过/最后）+ 时间/数量 + 后果暗示 | 禁止复制原文 |
| 信任建立 | 证据类型（面料/工艺）+ 验证方式（测试/对比）+ 保障 | 禁止复制原文 |

### 6. 促单闭环

从知识库中提炼：
- **尺码建议**：如何用身高体重对应降低决策成本
- **卡码建议**：如何处理用户犹豫区间
- **售后保障**：如何用确定性表达消除顾虑
- **临门一脚**：如何用紧迫感收尾

**促单闭环结构**：
```
痛点识别 → 方案提供 → 信任强化 → 决策降低 → 紧迫收尾
```

## 学习执行流程

### 第一步：整体感知

1. 通读知识库内容，形成整体印象
2. 标记所有引起注意的表达、结构和节奏
3. 识别知识库视频的核心卖点展示逻辑

### 第二步：结构拆解

按以下维度拆解知识库内容：

```
[黄金三秒]
- 钩子类型：________
- 情绪词：________
- 利益点：________

[塑品逻辑]
- 第一卖点：________
- 第二卖点：________
- 第三卖点：________

[促单话术]
- 决策引导：________
- 紧迫收尾：________
```

### 第三步：逻辑提炼

将拆解结果转化为可复用的逻辑：

| 知识库原文 | 核心逻辑 | SRT 筛选方向 |
|-----------|---------|-------------|
| "这个面料..." | 触感验证 | 在 SRT 中找面料描述 |
| "穿了三年..." | 耐穿背书 | 在 SRT 中找耐久性表达 |
| "110斤以下..." | 决策引导 | 在 SRT 中找尺码建议 |

### 第四步：SRT 筛选

带着提炼的逻辑回到当前 SRT：
1. 在 SRT 中寻找具有相同逻辑的原句
2. 只使用 SRT 原句，不复制知识库原文
3. 验证 SRT 原句是否符合知识库揭示的高转化模式

## 禁止行为

❌ **绝对禁止**：
- 直接复制知识库中的任何原文
- 将知识库话术改写后使用
- 用知识库表达替换 SRT 内容
- 声称知识库内容是"参考改编"实际上完全照搬

✅ **允许行为**：
- 学习"紧迫感"逻辑后在 SRT 中寻找类似原句
- 学习"痛点定向"后在 SRT 中筛选符合人群的原句
- 学习"塑品顺序"后在 SRT 中按相同逻辑排列片段

## 验证问题清单

在生成脚本前，必须自问：

1. 这句话是从 SRT 原文中选取的吗？
2. 我是否完全没有使用知识库原文？
3. 我是否只是学习了逻辑而不是复制了内容？
4. 知识库帮我理解了"什么样的句子更好"，但我没有直接拿来用？

如果任何答案不确定，必须重新检查。
