# 抖音带货视频规则参考

## 信息流分发适配规则

成片虽源自直播录屏，但最终用于**信息流长期分发**，因此必须适配以下规则。

### 话术适配要求

| 直播场景 | 信息流场景 | 转换要求 |
|---------|-----------|---------|
| 依赖即时库存 | 无库存压力 | 删除库存倒计时话术 |
| 依赖直播间互动 | 无互动场域 | 删除"扣1""评论区"等 |
| 依赖主播即时动作 | 无主播画面 | 删除动作指令话术 |
| 依赖物流即时承诺 | 无即时发货要求 | 改为通用发货承诺 |

### 禁止使用的话术类型

#### 1. 强实时直播互动话术

**定义**：明显依赖直播当下时点、库存瞬时变化、主播即时口播动作或直播间互动场域才能成立的话术。

**绝对禁止列表**：

| 话术类型 | 示例 | 禁止原因 |
|---------|------|---------|
| 库存倒计时 | "最后50件""仅剩100件""手慢无" | 触发低质录屏风险 |
| 上链接指令 | "来321上链接""现在去刷新" | 信息流场景违和 |
| 互动引流 | "扣1""扣222""来评论区" | 脱离互动场域无意义 |
| 即时物流 | "已经在仓不用等""现在下单马上发" | 依赖直播临场性 |
| 直播专属动作 | "来大家看我的手""手上有" | 无画面支撑 |
| 实时库存变化 | "刚刚又没了30件""库存告急" | 虚假信息风险 |

#### 2. 其他违规话术

| 类别 | 示例 | 违规原因 |
|------|------|---------|
| 极限词 | "最好""第一""国家级" | 违反广告法 |
| 虚假承诺 | "保证瘦""保证白""无效退款" | 夸大宣传 |
| 疗效承诺 | "治疗""治愈""止咳""消炎" | 医疗违规（除非有资质） |
| 低俗擦边 | 挑逗性描述/暗示性表达 | 内容违规 |
| 民族歧视 | 涉及民族的负面表达 | 政治敏感 |
| 虚假原价 | 虚构从未成交的高价 | 价格欺诈 |

## 内容合规标准

### 脚本合规检查

生成脚本时必须检查：

```
[ ] 是否含有库存倒计时话术
[ ] 是否含有上链接/刷新指令
[ ] 是否含有互动引流话术
[ ] 是否含有极限词/虚假承诺
[ ] 是否含有其他品牌名（除非是合作品牌）
[ ] 是否含有与商品品类不符的表达
```

### 发布文案合规

发布文案必须遵守：

1. **禁止极限词**：
   - 最高级："最好""第一""顶级""极品"
   - 虚假对比："秒杀所有""碾压同行"

2. **禁止虚假宣传**：
   - 价格虚构："原价999现价99"（必须有真实原价依据）
   - 功效夸大："立竿见影""永久有效"

3. **禁止医疗承诺**（非医疗商品）：
   - "治疗"_"改善""消炎""杀菌"等医疗词汇

### 商品品类相关性规则

**定义**：用户在当前对话中提供的商品标题所明确的品类（如裙子、裤子、上衣等）。

**校验要求**：

| 商品品类 | 允许的台词 | 禁止的台词 |
|---------|-----------|-----------|
| 裙子 | "这条裙子""裙子下摆""裙摆" | "这条裤子""裤腿""上衣口袋" |
| 上衣 | "这件上衣""袖子""领口" | "这条裤子""裙摆""下装" |
| 套装 | "套装设计""上下搭配" | 明确提及套装以外品类 |
| 配饰 | "搭配这个""配饰效果" | 提及服装品类 |

**执行要求**：
1. 若用户在当前对话中提供了商品标题，必须逐条检查所选台词是否与当前商品品类相符
2. 若台词内容明显与当前商品品类不符，一律剔除，不得进入脚本表格

## 视频技术规范

### 分辨率与格式

| 平台 | 推荐分辨率 | 宽高比 | 格式 |
|------|-----------|--------|------|
| 抖音 | 1080×1920 | 9:16 | MP4 |
| 快手 | 1080×1920 | 9:16 | MP4 |
| 视频号 | 1080×1920 | 9:16 | MP4 |

### 技术参数要求

| 参数 | 推荐值 | 说明 |
|------|-------|------|
| 码率 | 8000-15000 kbps | 越高越清晰但文件越大 |
| 帧率 | 30fps | 抖音支持 60fps |
| 编码 | H.264/H.265 | 兼容性最佳 |
| 音频 | AAC 192kbps | 确保听感清晰 |
| 文件大小 | ≤500MB | 单条视频建议控制 |

### FFmpeg 兼容性参数

```bash
# 移动端优化
-movflags +faststart

# 编码器选择
-c:v libx264 -crf 18 -preset fast
-c:a aac -b:a 192k

# VFR 处理（抽帧时必须）
-fps_mode vfr
```

## 完播率优化建议

### 时长控制

| 完播率目标 | 推荐时长 | 倍速选择 |
|-----------|---------|---------|
| 30-40% | 60-90秒 | 1.21 |
| 40-50% | 50-70秒 | 1.23 |
| 50%+ | 45-60秒 | 1.25 |

### 节奏优化

```
0-3秒：黄金开场，必须有钩子
3-10秒：核心卖点展示
10-30秒：价值强化+场景代入
30-50秒：痛点解决+信任建立
50-70秒：促单收尾+紧迫感
```

### 画面质量要求

- 主体清晰，避免过度抖动
- 光线充足，避免逆光/过曝
- 字幕与画面同步，无乱码
- 品牌水印位置合理，不遮挡主体
