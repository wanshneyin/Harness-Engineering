# 脚本表格模板参考

## 标准输出表格格式

执行脚本重组时，必须按以下格式输出表格：

### 表格字段说明

| 字段名 | 说明 | 示例 |
|--------|------|------|
| **脚本序号** | 当前片段的序号，从 1 开始递增 | 1, 2, 3... |
| **SRT 原序号** | 当前片段使用的原始 SRT 序号，必须连续 | `[1,2,3,4,5]` 或 `[1-5]` |
| **精确时间戳区间** | 包含"总区间"和"每条子区间"两层信息 | 见下方示例 |

## 时间戳格式规范

### SRT 标准格式
```
1
00:00:18,933 --> 00:00:19,866
第一句台词

2
00:00:19,866 --> 00:00:21,000
第二句台词
```

### 转换为 FFmpeg 秒数格式
- SRT 格式：`00:00:18,933` → FFmpeg 格式：`18.933`
- 计算方式：小时×3600 + 分钟×60 + 秒 + 毫秒/1000

### 转换示例

| SRT 格式 | FFmpeg 格式 |
|----------|-------------|
| 00:00:18,933 | 18.933 |
| 00:00:19,866 | 19.866 |
| 00:01:05,500 | 65.500 |
| 00:15:30,200 | 930.200 |

## 序号连续性校验

### 合法的合并情况

✅ **连续序号合并**（允许）：
- `[1,2,3,4,5]` - 完全连续
- `[6,7,8]` - 完全连续
- `[1-5]` - 使用连字符简写

❌ **非法的合并情况**（禁止）：
- `[1,2,4,5]` - 跳过了序号 3
- `[1,2,3,5]` - 跳过了序号 5
- `[1,3,5]` - 非连续序号混合

### 校验方法

在输出表格前，必须验证：
1. 所有 SRT 原序号是否为连续序列
2. 总区间的起点是否为序号首条的开始时间
3. 总区间的终点是否为序号末条的结束时间

## 语义闭合校验

### 完整语义示例

✅ **语义闭合**（可以截取）：
```
"这个价格真的不赚钱给大家送福利"
```
→ 句子完整，表达清晰

✅ **语义闭合**（可以截取）：
```
"老板说错过这次不会再有了"
```
→ 句子完整，有完整的意思表达

❌ **语义未闭合**（禁止截取）：
```
"今天给大家..."
```
→ 后文仍在下一条 SRT 中

❌ **语义未闭合**（禁止截取）：
```
"只要..."
```
→ 价格在后文 SRT 中，未闭合

❌ **语义未闭合**（禁止截取）：
```
"来我们看一下这个..."
```
→ 宾语"这个"后面未说明是什么

### 语义闭合决策树

```
当前片段结尾是否形成完整语义？
├── 是 → 检查时长是否 ≤8 秒
│         ├── 是 → 可以使用
│         └── 否 → 退回上一个完整语义点
└── 否 → 继续并入下一条连续 SRT
          └── 检查合并后是否 ≤8 秒
                ├── 是 → 重新判断语义闭合
                └── 否 → 放弃该候选片段
```

## 台词展开格式

### 正确格式

```
1. [SRT1原句内容]
2. [SRT2原句内容]
3. [SRT3原句内容]
```

### 错误格式

❌ 只显示部分台词：
```
1. [SRT1原句内容]
2. [SRT2原句内容]
```
→ 漏掉了 SRT3

❌ 概括改写：
```
这段讲的是面料优势和价格优惠
```
→ 必须保留原句

❌ 压缩合并：
```
这个面料超级好价格也不贵快来买
```
→ 必须逐条展开
