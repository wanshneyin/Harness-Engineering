### 3.3 消息过滤公共判断（互通群消息可见性规则）

```
┌──────────────────────────────────────────────────────────┐
│  消息进入过滤管线（统一入口）                                 │
│  filterSessionMsgs(msgs)                                 │
│  → 调用 store.getters['groupChat/filterMsgs'](msgs)       │
│  → 实际执行 msgFilter.js 的 filterMsgs()                   │
└──────────┬───────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────┐
│  filterMsgs(msgList, options)                            │
│  原逻辑：                                                 │
│  ① 屏蔽名单过滤 (shieldUserMids)                          │
│  ② 按 msg.type 分发到 msgFilterMap                        │
│     text → filterTextMsg                                  │
│     custom → filterCustomMsg                              │
│     notification → filterNotificationMsg                  │
│     tip → filterTipMsg                                    │
│     video → filterVideoMsg                                │
│                                                          │
│  【新增】options 扩展：                                    │
│  {                                                       │
│    isInteropGroup: Boolean,    // 当前群是否互通            │
│    isInteropEnabled: Boolean,  // 总开关                   │
│    currentUserRegion: 'domestic' | 'international',       │
│  }                                                       │
└──────────┬───────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────┐
│  filterCustomMsg 改造（核心变更）                           │
│                                                          │
│  在原有 switch(scene) 逻辑之前，插入互通过滤判断：           │
│                                                          │
│  if (filterCtx.isInteropGroup && filterCtx.isInteropEnabled) { │
│    const interopResult = filterInteropCustomMsg(msg, filterCtx) │
│    if (interopResult !== undefined) return interopResult  │
│    // undefined 表示不属于互通特殊规则，继续走原逻辑          │
│  }                                                       │
└──────────┬───────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────┐
│  filterInteropCustomMsg(msg, ctx)   【新增模块】           │
│  @file interopFilter.js                                   │
│                                                          │
│  const content = safeParseJSON(msg.content, {})           │
│  const msgType = content.type                             │
│  const region = ctx.currentUserRegion                     │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 规则1: 双端都不显示的卡片类型                          │  │
│  │ HIDDEN_BOTH = [                                    │  │
│  │   SHARE_ACTIVITY(23),    // 周年庆活动分享            │  │
│  │   RISK_TIPS_MSG(31),     // 交易私聊物品真实性提醒     │  │
│  │   SHARE_WANT_TO_BUY(18), // 求购分享                 │  │
│  │   SMALL_GAME(22),        // 小游戏分享                │  │
│  │   CREATE_VOTE_MSG(28),   // 群聊投票                  │  │
│  │   SMART_ANSWER_MSG(32),  // 国际群聊回答特定问题       │  │
│  │ ]                                                  │  │
│  │ if HIDDEN_BOTH.includes(msgType) → return false     │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 规则2: 大陆端显示、国际端不显示                        │  │
│  │ HIDDEN_INTERNATIONAL = [                           │  │
│  │   SHARE_LIVE_ROOM(27),   // 直播间分享               │  │
│  │ ]                                                  │  │
│  │ if HIDDEN_INTERNATIONAL.includes(msgType)           │  │
│  │   && region === 'international'                     │  │
│  │   → return false                                   │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 规则3: 双端都显示                                    │  │
│  │ SHOWN_BOTH = [                                     │  │
│  │   +1折叠消息,                                       │  │
│  │   CREATE_GROUP_MSG(16), // 创建群聊系统消息            │  │
│  │   OPEN_SERVER_REMIND(29), // 开服消息                 │  │
│  │ ]                                                  │  │
│  │ if SHOWN_BOTH.includes(msgType) → return true       │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 规则4: 需要区分游戏上架平台                           │  │
│  │ GAME_SPECIFIC = [                                  │  │
│  │   SHARE_CLICK_TOOL_PLAN(24), // 连点器方案           │  │
│  │   SHARE_TRANSACTION(9),      // 交易分享             │  │
│  │   SHARE_POSTS(19),           // 帖子分享             │  │
│  │ ]                                                  │  │
│  │ if GAME_SPECIFIC.includes(msgType):                 │  │
│  │   gameId = content.gameId                           │  │
│  │   gamePlatform = 查询游戏上架平台(gameId)              │  │
│  │   if (region=domestic && gamePlatform=onlyInternational)│
│  │     → return false // 国内看不到只上国际的游戏          │  │
│  │   if (region=international && gamePlatform=onlyDomestic)│
│  │     → return false // 国际看不到只上国内的游戏          │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │ 规则5: 群聊分享特殊处理                               │  │
│  │ if msgType === SHARE_GROUP_CHAT(15):                │  │
│  │   ① 同规则4的游戏平台判断                             │  │
│  │   ② 额外: 分享的目标群若非互通群 && 当前是国际端        │  │
│  │      → return false (国际端不显示非互通群的分享)        │  │
│  └────────────────────────────────────────────────────┘  │
│                                                          │
│  return undefined // 不属于互通特殊规则，走原有逻辑          │
└──────────────────────────────────────────────────────────┘
```

### 3.4 消息组件译文处理

```
┌──────────────────────────────────────────────────────────┐
│  消息渲染入口                                              │
│  MsgModel.vue → MsgBubbleContent.vue → 各 Msg*.vue        │
└──────────┬───────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────┐
│  判断当前群是否互通群                                       │
│  isInteropGroup = store.getters['groupChat/isInteropGroup']│
│                   (currentGroupTid)                       │
│                                                          │
│  如果不是互通群 → 走原有逻辑，不做任何修改                    │
│  如果是互通群 → 进入互通译文处理分支 ↓                       │
└──────────┬───────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────────────┐
│  场景1: 国际用户发消息 → 国内用户接收                               │
│                                                                  │
│  后端流程: 国际用户发送 → 后端翻译成中文 → 易盾/云信审核 → 通过后分发  │
│                                                                  │
│  前端处理:                                                        │
│  ① msg.custom 中后端写入:                                         │
│     senderRegion: 'international'                                 │
│     translateText: '中文译文'  (中文翻译)                            │
│     hasTranslation: 1                                             │
│                                                                  │
│  ② 国内用户端 (isForeign() === false):                             │
│     ┌──────────────────────────────────────────┐                 │
│     │ if (hasTranslation && translateText)       │                 │
│     │   → 显示 translateText（中文译文）           │                 │
│     │ else                                      │                 │
│     │   → 不显示该消息（中文未生成则国内不可见）    │                 │
│     │                                           │                 │
│     │ 注意: 需求要求"中文生成了国内才能接收到"      │                 │
│     │ → 实际由后端控制分发，前端做兜底判断           │                 │
│     └──────────────────────────────────────────┘                 │
│                                                                  │
│  ③ 卡片消息同理:                                                  │
│     content.translateData 中存放翻译后的字段                        │
│     国内端优先使用 translateData.fieldName                          │
│     备注字段(remark)对国内用户不显示                                 │
└──────────┬───────────────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────────────┐
│  场景2: 国内用户发消息 → 国际用户接收                               │
│                                                                  │
│  前端处理:                                                        │
│  ① 国际用户端 (isForeign() === true):                              │
│     ┌──────────────────────────────────────────┐                 │
│     │ if (msg.translateText 有值)                │                 │
│     │   → 显示译文 translateText                 │                 │
│     │ else                                      │                 │
│     │   → 显示原文 + 显示翻译按钮 🔄              │                 │
│     │   （点击翻译按钮 → translateSingleMsgV2）   │                 │
│     └──────────────────────────────────────────┘                 │
│                                                                  │
│  ② MsgBubbleContent.vue 的 showTranslate 计算属性需要调整:        │
│     现有逻辑: isForeign && flow=in && translatableType && isFail==1│
│     【新增】: 互通群内，国内→国际的消息如果无译文，也要显示翻译按钮   │
│                                                                  │
│     调整后:                                                       │
│     showTranslate() {                                            │
│       if (!this.isForeign) return false                          │
│       if (this.source?.flow === 'out') return false              │
│       if (!translatableTypes.includes(type)) return false        │
│                                                                  │
│       // 原有逻辑: 翻译失败时显示                                  │
│       if (this.source.isFail == 1) return !this.translating      │
│                                                                  │
│       // 【新增】互通群内无译文时也显示翻译按钮                      │
│       if (this.isInteropGroup && !this.source.translateText)     │
│         return !this.translating                                 │
│                                                                  │
│       return false                                               │
│     }                                                            │
└──────────┬───────────────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────────────┐
│  场景3: 国内用户发消息 → 国内用户接收                               │
│                                                                  │
│  前端处理:                                                        │
│  ① 显示原文，无需翻译                                              │
│  ② 【新增】MsgModel / MsgUserHeader 中:                           │
│     如果当前群是互通群 && 发送者 senderRegion === 'domestic':       │
│       → 用户名旁显示 "国内" 标签                                   │
│     注意: 此标签需后门开关控制（开发自行在后门给开关）                 │
└──────────┬───────────────────────────────────────────────────────┘
           ▼
┌──────────────────────────────────────────────────────────────────┐
│  列表页摘要处理                                                    │
│  @file GroupItem.vue                                              │
│                                                                  │
│  现有逻辑: displayMsg 根据 lastMsg 生成摘要文案                     │
│                                                                  │
│  【新增】互通群特殊摘要:                                           │
│  ┌───────────────────────────────────────────────────────┐       │
│  │ if (isInteropGroup && isForeign() === false) {         │       │
│  │   // 国内端接收到国际用户的纯文本消息                     │       │
│  │   if (lastMsg.senderRegion === 'international'         │       │
│  │       && lastMsg.type === 'text') {                    │       │
│  │     displayText = `${senderNick}：发来一条消息`          │       │
│  │     // 多语言 key: 46669 / 46670                       │       │
│  │   }                                                    │       │
│  │ }                                                      │       │
│  └───────────────────────────────────────────────────────┘       │
│                                                                  │
│  同时，群聊消息通知也走此逻辑:                                      │
│  系统消息通知文案: "群聊 {groupId} 发来一条消息"                     │
└──────────────────────────────────────────────────────────────────┘
```
